Platform Guide

AssayTrace Documentation

How to run a change-control analysis, interpret revalidation decisions, and produce audit-ready outputs from your assay manifests and supporting evidence.

Inputs

Baseline Manifest

The validated, currently deployed assay manifest (JSON). Serves as the reference configuration for change detection.

Candidate Manifest

The proposed updated assay manifest (JSON). AssayTrace diffs this against the baseline to identify every versioned change.

Benchmark Evidence

Optional baseline and current benchmark JSON packages (e.g. GIAB-derived metrics). Used for performance regression detection alongside manifest changes.

Reportable Variants

Optional old and new reportable variant JSON lists. Enables classification concordance and variant delta reporting between pipeline versions.

Decision Types

classification_concordance_review

Classification Concordance Review

Recommended when interpretive resources change (e.g. annotation or transcript updates) and clinical classification consistency must be verified across reportable variants.

targeted_analytical_revalidation

Targeted Analytical Revalidation

Recommended when analytical components change in ways that may affect validated detection performance. Scope is limited to the impacted claims and variant types.

infrastructure_verification

Infrastructure Verification

Recommended when container images, workflow engines, or deployment environment metadata change. Confirms the runtime environment remains controlled and traceable.

qc_verification

QC Verification

Recommended when QC thresholds or QC-related policies change. Verifies that quality decision boundaries remain appropriate and documented.

Outputs

  • Change Summary — tabular record of every detected manifest difference with old and new values.
  • Impact Summary — impact domain and rationale for each change.
  • Decision Rationale — revalidation type, affected claims, and explanation for each decision.
  • Regression Analysis — baseline vs. current benchmark metrics with regression status when benchmark evidence is provided.
  • Variant Delta Report — added, removed, and classification-changed reportable variants when variant JSON is provided.
  • Audit Binder PDF — downloadable, audit-ready PDF assembled from the full analysis record.

FAQ

What is a manifest?

An AssayTrace manifest is the canonical identity card of a clinical NGS assay pipeline. It captures versioned analysis components, reference resources, QC thresholds, validated assay claims, and environment metadata in a single structured, hashable document.

Why is benchmark evidence useful?

Manifest changes describe what changed; benchmark evidence shows whether performance changed. Comparing baseline and current benchmark packages helps detect analytical regressions that may require follow-up even when the manifest diff alone does not capture runtime behavior.

When is revalidation required?

Revalidation is recommended when a change affects validated assay claims in an analytical, interpretive, quality, or infrastructure domain. AssayTrace maps each change to a specific revalidation type. Changes that require only documentation updates or no action are also reported explicitly.

What is an Audit Binder?

The Audit Binder is the complete change-control record produced by AssayTrace: detected changes, impact classifications, claim mappings, revalidation decisions, benchmark regression results, variant deltas, and sign-off metadata — suitable for quality review and PDF export.
