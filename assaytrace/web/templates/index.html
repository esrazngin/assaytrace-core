Change Control · Revalidation · Auditability

Know what changed between two NGS workflow versions — and what revalidation it requires.

AssayTrace compares two versions of a clinical NGS assay manifest (software, references, parameters, panel, QC, environment, and the performance claims that depend on them), classifies the impact of each change, maps it to affected claims, and produces a defensible revalidation decision with audit-ready evidence.

Who it's for: bioinformaticians, clinical genomics labs, and QA/regulatory teams operating under CLIA / CAP / IVDR who need traceable change control across assay versions.

What it does

Manifest diffing → impact classification → claim-dependency tracking → revalidation guidance → audit binder (HTML/PDF).

What AssayTrace does NOT do
  • It does not run NGS pipelines or execute workflows.
  • It does not analyze FASTQ / BAM / VCF sequencing data.
  • It does not call, classify, reinterpret, or clinically interpret variants — the Variant Delta only compares result sets you supply.
  • It is not a medical device and does not issue clinical determinations.

Supported Inputs & Formats

Assay ManifestJSON / YAML
The core input. A structured description of one assay version. Two are compared (old vs new). Required.
Software Componentsin manifest
Aligner, variant/CNV/SV callers, filters — name, version, and parameters.
Reference Resourcesin manifest
Reference genome build/version and target BED/panel definitions.
Annotation Databasesin manifest
ClinVar, gnomAD, COSMIC, transcript sets — referenced by version, not re-run.
Environment Metadatain manifest
Container runtime, image digest, workflow engine, build provenance.
Benchmark & VariantsJSON (optional)
GIAB precision/recall/F1 for regression, and reportable-variant lists for classification deltas.
